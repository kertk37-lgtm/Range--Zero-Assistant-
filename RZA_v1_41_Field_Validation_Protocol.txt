RZA v1.41 FIELD VALIDATION PROTOCOL
Controlled Workflow Refinement Build

Purpose
Confirm that the v1.41 app remains accurate and stable when used with printed validation targets, real photos, APP ALIGN, manual shot marking, flyer exclusion, and saved session packages.

Printer setup for all targets
1. Print at 100% / Actual Size.
2. Disable Fit to Page / Scale to Fit.
3. Use 8.5 x 11 inch paper unless the target PDF specifies otherwise.
4. After printing, measure a known 1-inch grid span or fiducial distance if available. Do not continue with a scaled print.

TARGET SET 1 - Page 6 Exact APP ALIGN Validation Suite v5
File: RZA_Page6_Exact_APP_ALIGN_Validation_Suite_v5.pdf
Purpose: Known-answer app/math confirmation.

Step 1: Open the Page 6 validation suite from inside the app.
Step 2: Print at 100% actual size.
Step 3: Use 2-point APP ALIGN first.
Step 4: Run the known-answer test cases A-D.
Step 5: Confirm that the app output matches the printed expected correction values.
Step 6: Save a session package for each case.

Pass standard:
- Case A matches expected correction.
- Case B matches expected correction.
- Case C matches expected correction.
- Case D matches expected correction when the flyer is excluded.
- Zero-axis output displays plain 0 instead of 0 LEFT / 0 RIGHT / 0 UP / 0 DOWN.

TARGET SET 2 - v1.37 Enhanced APP ALIGN Fiducial Target Pack
File: RZA_v1_37_APP_ALIGN_Fiducial_Target_Pack.pdf
Purpose: Real-photo 4-point APP ALIGN geometry confirmation.

Step 1: Print at 100% actual size.
Step 2: Photograph the target square-on.
Step 3: Photograph again from a mild angle.
Step 4: Photograph again from a moderate angle.
Step 5: Photograph again from a stronger but still usable angle.
Step 6: In the app, use 4-point APP ALIGN: TL, TR, BL, BR.
Step 7: Mark the same known points or test impacts each time.
Step 8: Compare output across photo angles.

Pass standard:
- Direction remains stable across all usable photos.
- Click magnitude remains close enough for operational use.
- No high/low inversion occurs.
- No left/right inversion occurs.
- Strongly distorted photos may be rejected operationally even if the math still resolves them.

TARGET SET 3 - Page 6 Large Format Duplex Target
File: RZA_Page6_Large_Format_Duplex_Target.pdf
Purpose: Larger-format workflow confirmation and repeatability.

Step 1: Print at 100% actual size.
Step 2: Use a clean target photo and confirm it as clear/correct.
Step 3: Use a marked/final target photo.
Step 4: Run the workflow in the intended order: clean photo, final photo, APP ALIGN, aim point, shots, calculate, save.
Step 5: Repeat the same target/shot marks three times.

Pass standard:
- Repeated marking gives the same or nearly the same correction.
- Saved session packages preserve shot sequence, APP ALIGN points, aim point, recommendation, and diagnostic data.
- The diagnostic overlay visually agrees with the calculated group center.

TARGET SET 4 - Page 9 Large Format Duplex Target
File: RZA_Page9_Large_Format_Duplex_Target.pdf
Purpose: Multi-zone / selected-zone workflow confirmation.

Step 1: Print at 100% actual size.
Step 2: Select the correct page/zone in the app.
Step 3: Use one zone for Group 1.
Step 4: Save the result.
Step 5: Use Apply & Start Next Group.
Step 6: Move to a fresh zone for Group 2.
Step 7: Save/export the session data.

Pass standard:
- Group number increments properly.
- Previous group remains saved.
- New group starts clean.
- Zone/page metadata is preserved in export.

TARGET SET 5 - Live-fire or dry known-offset confirmation
File: Use the best matching printed RZA target for the range session.
Purpose: Physical confirmation of theory.

Step 1: Start at 25 or 50 yards for conservative confirmation.
Step 2: Fire or mark a controlled 3-5 shot group.
Step 3: Run the app recommendation.
Step 4: Apply the recommended adjustment or simulate applying it.
Step 5: Fire or mark a second group on a fresh target zone.
Step 6: Confirm the recommendation trends toward 0.
Step 7: Continue until both axes display 0 or a practical zero-confirmation state.

Pass standard:
- Corrections move in the expected direction.
- Groups converge toward point of aim.
- 0/0 state is understandable and operationally useful.
- No unexplained direction reversal occurs.

Minimum acceptance standard for v1.41
The build is acceptable for GitHub Pages testing if:
1. The embedded known-answer validation suite passes.
2. 2-point APP ALIGN and 4-point APP ALIGN do not invert correction direction.
3. Zero-axis output displays cleanly as 0.
4. Extreme-spread group size is reported instead of centroid-radius estimate.
5. Saved sessions preserve the data needed to audit the result.
6. Export data works and contains setup/session information.
7. No broken local links or missing packaged assets are found.

Recommended next decision after field validation
If the above passes, v1.41 can remain the stable validation baseline. The next refinement should be v1.42 and should focus on validation UX, confidence checks, or stronger user guidance rather than changing the math core.
